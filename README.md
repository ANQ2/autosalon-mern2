# CarLine

CarLine — full-stack веб-приложение для управления автосалоном.  
Проект разработан в рамках учебной работы и моделирует основные процессы работы автосалона с разделением прав доступа между пользователями.

Приложение реализовано на стеке MERN с использованием GraphQL, JWT-аутентификации и монорепозитория.

---

## Описание проекта

Цель проекта — разработать веб-систему автосалона, приближенную к реальным коммерческим решениям.  
В отличие от простых CRUD-приложений, в проекте реализованы:

- ролевая модель пользователей;
- защита маршрутов;
- разделение frontend и backend;
- real-time обновление данных.

Проект демонстрирует навыки full-stack разработки и понимание архитектуры веб-приложений.

---

## Основной функционал

- Регистрация и авторизация пользователей
- Ролевая модель доступа
- Просмотр каталога автомобилей
- Управление автомобилями (для менеджеров)
- Назначение ролей (для администратора)
- Защита маршрутов на клиенте и сервере
- Обновление данных без перезагрузки страницы

---

## Роли пользователей

### CLIENT
- Просмотр списка автомобилей
- Доступ только к публичной информации
- Нет прав на изменение данных

### MANAGER
- Создание и управление автомобилями
- Работа с бизнес-логикой автосалона
- Доступ ограничен менеджерским функционалом

### ADMIN
- Назначение ролей пользователям
- Полный контроль доступа
- Управление безопасностью системы

## Архитектура проекта

Проект реализован в формате **монорепозитория**, где клиентская и серверная части находятся в одном GitHub-репозитории.

carline/
├── client/ # Frontend (Next.js)
├── server/ # Backend (Node.js + GraphQL)
└── README.md


### Frontend
- Next.js (App Router)
- Tailwind CSS
- Zustand
- Apollo Client

### Backend
- Node.js
- Express
- GraphQL (queries, mutations, subscriptions)
- JWT-аутентификация
- Ролевая модель доступа (RBAC)

### База данных
- MongoDB
